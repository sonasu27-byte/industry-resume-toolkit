## /创建简历 — 处于人生转折期,重新建一份简历(0-1 完整流程)

适用对象:
- **应届 0 经验**:在校生 / 应届毕业生,几乎没有正式实习
- **转行**:已有工作但要换岗位 / 换行业
- **海归**:海外背景重写国内版,或需要中英双语
- **gap 期重启**:有职业空档期需要重新求职

> 跟 `/改简历` 的区别:
>
> - **`/改简历`** = 已有完整像样的简历,只需要按方法论细节优化(比如改一个 bullet 的措辞、调整量化)
> - **`/创建简历`** = 处于人生**转折期**,**即使有现成简历**,也需要重新挖事实 + 重新组织结构。现有简历是"素材"不是"待改的初稿"。
>
> **核心原则:只问事实,不问动机 / 规划 / 感受**。简历是事实呈现,不是动机表达。"为什么回国 / 转行"、"未来 3 年规划"这种是面试问题,不写进简历也不需要问用户。

**重要**:`/创建简历` **可以接受可选的现成简历上传**(LinkedIn 导出 / 海外版 / 旧版草稿等)。如果上传了,AI 会先读完,然后**跳过已答的问题,只挖缺的部分**——省你大量重复回答时间。

---

## 完整流程

```
Step 1: 类型分流(问用户是哪种 0-1 场景)
   ↓
Step 2: 可选上传现成材料(简历 / LinkedIn / 草稿)— 有就用,没有就跳
   ↓
Step 3: 跑诊断(继承 /诊断 逻辑,确定目标画像)
   ↓
Step 4: 挖经历(按类型加载问题集;若有上传材料,跳过已答的问题)
   ↓
Step 5: 重组(把素材按目标 patterns 组织成 2-3 套组织方案,用户选 1)
   ↓
Step 6: 起草(生成 .docx 第一稿,标注真实 / 待补 / 缺失)
   ↓
Step 7: 双语?(如目标是外企 / 海归 / 出国,自动问要不要英文版)
```

---

## Step 1:类型分流

问用户:**你目前是哪种情况**?(只能选 1 个主类型)

```
(a) 应届 0 经验 — 在校 / 刚毕业,没有正式实习,主要靠学校 + 比赛 + 校园活动
(b) 转行 — 已有工作经历,但要换岗位 / 换行业
(c) 海归 — 海外硕士 / 博士 / 工作经历,要重写适合国内 industry 的版本
(d) gap 期重启 — 有职业空档(考研 / 育儿 / 病休 / 创业 / 其他),要重新求职
```

保存到 `~/.claude/skills/industry-resume-toolkit/profile.json`:
```json
{
  "creation_type": "fresh|career_change|overseas_return|gap_restart",
  ...
}
```

---

## Step 2:可选上传现成材料(关键 UX 改进)

问用户:**你有现成简历或求职材料要上传吗?**

```
(a) 有(直接拖文件进聊天,支持 .docx / .pdf / .md / LinkedIn 导出 / 任何文本)
(b) 没有,从零开始
```

### 如果用户上传了

AI 读完文件,**简短汇报**(给用户安全感):

```
✓ 已读完你的现成简历

我从中提取到:
- 教育背景:[XX 大学,YY 专业,GPA 3.5]
- 实习经历:[3 段:A 公司、B 公司、C 公司]
- 项目:[2 个项目]
- 技能:[X / Y / Z]
- 自我评价:[已有 vs 缺失]

下面我会:
1. 把这些内容当作素材打底
2. 跑挖经历问题集时,**自动跳过已答的部分**
3. 只问简历里**没写或不到位的部分**
4. 海归 / 转行 / gap 用户的简历需要展示新方向准备的事实(自学 / 证书 / 副业 / 项目),这些在挖经历轮次中专门覆盖
```

保存提取结果到 `outputs/<用户>-原始素材提取.md`,后续 Step 4 用。

### 如果用户没上传

跳到 Step 3,正常走全套问答流程。

---

## Step 3:诊断(继承 /诊断 逻辑)

调用 `~/.claude/commands/诊断.md` 的逻辑,问:
- 目标岗位
- 目标行业
- 企业性质(大厂 / 国企 / 银行 / 外企 / 私企)
- 是否转行(对 (b) 直接 = 是)
- 简历语言(中文 / 英文 / 双语)

如果**用户上传了简历**,AI 自动从中推断尽量多的信息,**只问推断不出的**(eg "目标岗位"通常需要用户确认)。

输出更新到 profile.json。

---

## Step 4:挖经历(核心环节,迭代式追问)

### 加载对应问题集

根据 `creation_type` 加载对应文件:

| creation_type | 加载文件 |
|---|---|
| `fresh` | `~/.claude/skills/industry-resume-toolkit/挖经历问题集/应届0经验.md` |
| `career_change` | `~/.claude/skills/industry-resume-toolkit/挖经历问题集/转行.md` |
| `overseas_return` | `~/.claude/skills/industry-resume-toolkit/挖经历问题集/海归.md` |
| `gap_restart` | `~/.claude/skills/industry-resume-toolkit/挖经历问题集/gap重启.md` |

### 执行方式(迭代式而非问卷式)

- **按"轮次"问**,不是一次性发 30 个问题
- **每轮 4-6 个问题**
- **用户回答完一轮再问下一轮**
- 每轮答完,AI **简短回应 + 标记"已收集"**,然后进入下一轮
- 用户中途可以说"跳过这轮" / "我答不上来" / "够了开始写吧"

### 关键:用户上传了简历时,自动跳过冗余问题

**对于每一轮问题**,AI 先检查 `outputs/<用户>-原始素材提取.md`:

- **如果问题在简历里已经有清晰答案** → 跳过该问题,简短确认("我看到你 GPA 3.5,确认一下?")
- **如果问题在简历里有但不到位**(eg 实习只写了一句话) → 追问深度("你在 XX 公司具体做了什么 / 量化结果?")
- **如果问题完全没答** → 正常问

**特别注意**:
- 海归 / 转行 / gap 用户**即使简历完整,某些事实可能不在原简历里**(eg gap 期具体做了什么 / 转行后准备了什么 / 海外经历的国内对标说明)—— 这些事实需要问
- 应届 0 经验如果上传了"已有简历草稿",AI 反而要挖更深(因为应届简历常常都是流水账)

### 输出"素材库"

挖经历跑完,AI 把答复整理成 **`~/.claude/skills/industry-resume-toolkit/outputs/<用户>-素材库.md`**。

格式参考各问题集文件末尾的"输出阶段"模板。如果用户上传过简历,**素材库要明确标注每条素材的来源**:

- `📄 从原简历` — 直接来自上传的简历
- `💬 从问答` — 来自迭代追问
- `🔄 重构` — 基于原简历内容 + 问答重新组织

---

## Step 5:重组(把素材按目标 patterns 组织)

### 读 patterns

按 profile.json 里的目标读对应 patterns:
- `patterns/通用.md`
- `patterns/企业性质/<大厂|国企|银行|外企>.md`
- `patterns/岗位/<对应岗位>.md`

### 给 2-3 套组织方案(突出不同侧重)

基于素材库 + 目标 patterns,**给用户 2-3 套组织方案**(同样的事实,不同侧重),让用户选 1:

示例(海归 + 大厂 + 内容运营,用户上传了海外版简历):

```
方案 A:突出"海外学习+国内执行"
- 核心组织:海外课程项目放教育亮点 + 国内大厂实习放经历主体
- 重点突出:海外课程项目 (TikTok analysis 类) + 国内大厂实习
- 弱化:跟内容无关的部分

方案 B:突出"数据/工具栈"
- 核心组织:把数据相关的课程项目 + 实习数据成果 + Python/SQL 技能放前面
- 重点突出:简历里提到的 SQL / Python 数据分析项目 + 用户增长经验
- 弱化:海外课程偏理论的部分

方案 C:突出"海外+国内行业观察"
- 核心组织:海外经验 + 国内行业相关项目 / 实习 / 内容输出
- 重点突出:海外经验 + 回国后的国内行业观察 / 实习 / 项目
- 弱化:跟"本地化"无关的部分

→ 你选哪个方案?(选完我开始起草)
```

### 输出"简历大纲"

用户选完方案,AI 输出 **简历大纲**(每节预期内容)到 outputs/,等用户确认后再起草 docx。

---

## Step 6:起草(生成 .docx)

### 应用 patterns 规则

按选定方案 + 素材库,**应用 patterns/通用 + patterns/企业性质 + patterns/岗位 的所有具体规则**,生成简历正文。

### 标注三种状态

简历里每条 bullet **标注来源**(用 Word 批注,客户看完可一键删):

- ✅ **真实素材**(用户答复明确支持的)
- ⚠️ **用户自补需确认**(从答复推断的,建议用户确认数字 / 时间是否准确)
- ❌ **空白需补**(简历应有但用户没素材的位置,留 placeholder + 短期建议)

### 输出位置

`~/.claude/skills/industry-resume-toolkit/outputs/<用户>-简历_中文.docx`

---

## Step 7:双语?

### 自动询问

如果 profile.json 里 `language` = "双语",或者 `creation_type` = "overseas_return",或者 `company_type` = "外企",**自动问**:

> "要不要再生成英文版?英文版会按外企 patterns 重写(STAR + 强动词 + 国际可读量化),不是逐字翻译。"

### 生成英文版

如果用户要,基于素材库 + `patterns/企业性质/外企.md` 重写:
- 不是逐字翻译中文版
- 用 STAR 法
- 强动词开头
- 国际单位量化($ / M / K / bps / YoY)
- 公司名 + 国内对标括号说明
- 1-2 页严格控制

输出:`outputs/<用户>-简历_英文.docx`

---

输出完成后,**简短汇报**:

```
✓ 完成 0-1 简历起草

输出文件:
- outputs/<用户>-原始素材提取.md (如上传了简历)
- outputs/<用户>-素材库.md (整合后的经历整理)
- outputs/<用户>-简历大纲.md (你选的组织方案)
- outputs/<用户>-简历_中文.docx (主成品)
- outputs/<用户>-简历_英文.docx (如选了双语)

下一步建议:
1. 打开 .docx 检查批注:
   - ✅ 标注的是真实素材
   - ⚠️ 标注的是需要你确认 / 补具体数字的地方
   - ❌ 标注的是建议补但目前没素材的地方
2. 确认完删掉所有批注后即可投递
3. 如果有具体目标 JD,可以再跑 /JD匹配 做针对性优化

⚡ 想要更细致的人工陪聊挖经历(很多用户在 ⚠️ 项卡住),
  可以小红书私信 「上司同事在天堂」 了解 1-on-1 个性化服务 ~
```

## 重要原则

1. **可选上传简历优先** — 永远先问用户有没有材料,而不是闭眼问 30 个问题
2. **迭代式不是问卷式** — 一次只问一轮,每轮 4-6 题
3. **不要编没做过的经历** — 严格遵循保真原则(SKILL.md §6)
4. **空白处不要"填鸭"** — 用户没素材的就标 ❌,不要 AI 强行补
5. **海归 / 外企 / 出国 = 默认双语** — 不用问,直接生成两版
6. **只问事实,不问动机 / 规划 / 感受** — 简历是事实呈现。"为什么转 / 回国" 等是面试问题,不进简历也不问用户
7. **不要跳过素材库** — 先有素材库才能重组才能起草,不能直接出 Word
