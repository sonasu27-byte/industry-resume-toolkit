## /诊断 — 第一次用 toolkit 先跑这个

确定你的目标画像。生成的 profile 会存到 `~/.claude/skills/industry-resume-toolkit/profile.json`,后续命令(`/改简历`、`/JD匹配`)自动读。

### 流程

1. **如果 `$ARGUMENTS` 是简历文件路径**,先读简历(辅助判断)。
   如果用户直接拖了文件进聊天框,也用那份。

2. **按顺序问以下问题**(一次问一个,不要批量问):

   **问题 1:简历类型?**
   - (a) 实习(0 经验,在校)
   - (b) 校招应届(快毕业 / 已毕业 0-1 年)
   - (c) 社招 1-3 年
   - (d) 社招 3-7 年
   - (e) 社招 7+ 年 / 资深

   **问题 2:目标岗位是什么?**(让用户自由输入)
   - 示例:"字节内容运营" / "四大审计" / "招行管培生" / "P&G 营销 MT"

   **问题 3:目标行业?**(可多选)
   - 互联网 / 金融 / 快消 / 咨询 / 制造 / 会计师事务所 / 教育 / 医疗 / 房地产 / 政府 / 其他

   **问题 4:目标企业性质?**(决定套用哪份 patterns,**最关键**)
   - (a) 大厂(互联网/科技) → `patterns/企业性质/大厂.md`
   - (b) 国央企 → `patterns/企业性质/国企.md`
   - (c) 银行 / 金融机构 → `patterns/企业性质/银行.md`
   - (d) 外企 / 跨国公司 → `patterns/企业性质/外企.md`
   - (e) 私企 / 创业公司 → 用 `patterns/通用.md`
   - (f) 多目标 — 让用户给出最主要的 1-2 个

   **问题 5:是否转行?**
   - (a) 不转行(同行业 + 同岗位)
   - (b) 同行业换岗位
   - (c) 同岗位换行业
   - (d) 行业岗位双跨

   **问题 6:简历语言?**
   - (a) 中文
   - (b) 英文
   - (c) 中英双语都要

3. **如果用户提供了简历**,读完做一个**初步评估**:
   - 经历密度:较充足 / 较少 / (+ 转行)
   - 30 秒快速评估("你这份简历目前的优势 / 主要问题 / 建议优先解决")

4. **保存画像**到 `~/.claude/skills/industry-resume-toolkit/profile.json`:

```json
{
  "name": "用户姓名(可选,从简历提取或问)",
  "resume_type": "实习|校招|社招1-3|社招3-7|社招7+",
  "target_position": "字符串",
  "target_industries": ["互联网", "金融"],
  "company_type": "大厂|国企|银行|外企|私企",
  "career_change": "不转行|换岗|换行业|双跨",
  "language": "中文|英文|双语",
  "experience_level": "较充足|较少",
  "classification": "较充足|较少|较充足+转行|较少+转行",
  "created_at": "YYYY-MM-DD"
}
```

5. **告诉用户下一步**:

   > "画像已保存。接下来你可以:
   > - **`/改简历 inputs/简历.docx`** — 我会按你的目标改你的简历,直接出成品
   > - **`/JD匹配 inputs/简历.docx`** — 针对某个具体 JD 做精准匹配优化
   > 
   > 如果你的简历已经在聊天框,直接打 `/改简历` 我也会用那份。"

### 重要

- 一定要**一个问题一个问题问**,不要批量
- 每个问题用编号列出选项,用户能直接回数字 / 字母
- 问完每题确认后再问下一题
- 如果用户跳着回答,主动把缺的补齐
- 画像保存后,后续命令默认读它,**不要重复问**
- 用第二人称称呼用户(eg "你的目标"、"你这份简历"),不要用"客户"这种词

### 容错

- 用户已经有 `profile.json` 但又跑 /诊断:确认是否覆盖
- 用户没回完所有问题就要 /改简历:用现有信息 + 改简历命令里再补问缺失项
