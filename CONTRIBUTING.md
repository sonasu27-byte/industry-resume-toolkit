# Contributing to Industry Resume Toolkit

感谢你想为这个 toolkit 贡献力量!以下是几种最有价值的贡献方式:

> **先了解授权方式**:本项目采用 [CC BY-NC-ND 4.0](LICENSE)(署名-非商业-禁止演绎)。这意味着**别人不能自行公开发布修改版或翻译版**。但社区贡献依然欢迎——所有改动都以 **PR 形式合并进本官方仓库**,由作者统一维护。fork 只用于给本仓库提 PR,**请不要拿 fork 去单独发布一个修改/翻译后的版本**。

## 最欢迎的贡献类型

### 1. 新岗位 patterns(优先)

目前 v0.1 已有 8 个:运营 / 产品 / 数据分析 / 市场营销 / 财务 / 审计 / HR / 销售。

缺的还有很多:
- **开发**(后端 / 前端 / 算法 / 测试 / 全栈 / 移动 / DevOps)
- **法务**(企业法务 / 律所律师 / 合规)
- **设计**(UI / UX / 视觉 / 产品设计 / 工业设计)
- **战略 / 咨询**(战略 / 投资分析 / 行业研究)
- **采购 / 供应链**(SCM / 物流 / 仓储 / 招采)
- **客服 / 客户成功**(CSM / 客服 / 售后)
- **行政 / 秘书**(行政管理 / 高管秘书)
- **公关 / 媒介**(企业传播 / 媒介关系)

每个新岗位 patterns 文件按 `patterns/岗位/产品.md` 的结构来写:
- 风格关键词
- 核心规则(6-8 条具体的)
- 不同企业性质下的岗位写法差异
- 常见地雷
- 校招 / 实习特别建议
- 自我评价写法
- 出处 

放到 `patterns/岗位/` 文件夹下,文件名用中文(eg `法务.md`)。

### 2. 新企业性质 patterns

目前已有:大厂 / 国企 / 银行 / 外企。缺的:
- **私企 / 创业公司**(独角兽 / 早期 / 中小型)
- **媒体 / 文娱**(电视台 / 报社 / 视频平台 / MCN)
- **医疗 / 制药**(MNC 药企 / 医院 / 医疗器械)
- **教培 / 教育**(K12 / 高教 / 留学机构 / 网课平台)
- **房地产 / 建筑**(地产开发 / 建筑设计 / 物业)
- **快消 / 零售**(品牌方 / 电商 / 连锁)

放到 `patterns/企业性质/` 文件夹下,按 `patterns/企业性质/大厂.md` 的结构来写。

### 3. 脱敏的真实案例

贡献你改过 / 看过 / 自己的简历 before/after 案例,**前提是完全脱敏**(姓名、公司、学校、具体可识别数据都改掉)。

格式参考 `examples/经验充足/` 和 `examples/经验较少/`。

通过审核的案例会加入 examples/ 文件夹。

### 4. 翻译

- README 翻译到其他语言(目前有中英)
- SKILL.md 翻译

> 翻译属于"演绎",请**以 PR 形式合并进本官方仓库**,成为官方多语言版本的一部分;请勿自行对外发布独立的翻译版本。

### 5. Bug 修复 / 改进

发现 patterns 里的规则错误、prompt 写法低效、命令逻辑 bug,都可以 PR。

## 怎么贡献

1. Fork 这个 repo
2. 创建你的 branch (`git checkout -b feature/new-pattern-xxx`)
3. 提交修改 (`git commit -m 'Add XXX pattern'`)
4. Push 到 fork (`git push origin feature/new-pattern-xxx`)
5. 开 Pull Request

### Patterns 文件的要求

- **必须有具体的、可执行的规则**(不要写"要重视质量"这种空话)
- **必须有示例**(原文 X → 改成 Y)
- **必须有"常见地雷"**(避免做错的事)
- **必须有"出处"标注**(从哪个案例提炼 / 行业惯例 / 来源)

### 案例文件的要求

- 完全脱敏:姓名、公司、学校、地址、邮箱、电话全部假名 / 删除
- 数据脱敏:具体的销售额、用户量等改成相对量级(eg "数百万 → X 万级")
- 提交时附说明:"已脱敏,无可识别信息"

## 贡献的授权约定

提交 PR 即代表你同意:

- 你的贡献是**你的原创**(或你有权提交),没有侵犯他人著作权;
- 案例类贡献已**完全脱敏**,不含任何可识别真人的信息;
- 你的贡献一经合并,将以本项目的 [CC BY-NC-ND 4.0](LICENSE) 授权对外发布,并**授权作者(上司同事在天堂)对其使用、合并、修改、维护**;
- 你保留作为贡献者的署名权,但不对合并后的整体作品另行主张商业权利。

## 不接受的贡献

- 推广广告 / 软文
- 抄袭其他工具的内容(不尊重 fellow 开源贡献者)
- 没有 README 解释的功能
- 跟核心方法论冲突的改动(eg 鼓励夸大职责,违反保真原则)
