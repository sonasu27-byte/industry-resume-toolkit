# Patterns 库结构

本 toolkit 的核心差异化护城河——按**企业性质 × 岗位**两层组织的规则库

## 两层结构

```
patterns/
├── 通用.md                    ← 跨企业、跨岗位的通用规则
├── 企业性质/                  ← 第一层维度:企业性质决定整体风格
│   ├── 大厂.md
│   ├── 国企.md
│   ├── 银行.md
│   └── 外企.md
└── 岗位/                      ← 第二层维度:岗位决定具体技能词
    ├── 运营.md                (含内容运营 / 用户运营 / 活动运营等)
    ├── 产品.md                (产品经理)
    ├── 数据分析.md
    ├── 市场营销.md
    ├── 财务.md                (含会计 / 出纳 / 财务分析 / FP&A 等)
    ├── 审计.md                (会计师事务所专项)
    ├── HR.md                  (含 HRBP / 招聘 / 培训 / 薪酬等)
    ├── 销售.md                (含 B 端销售 / KA / BD / CSM 等)
    ├── 开发.md                (含后端 / 前端 / 算法 / 数据工程等)
    ├── 法务.md                (含律所律师 / 企业法务 / 合规)
    ├── 战略咨询.md            (含 MBB / 战略部 / 投资战略 / 行研)
    └── 设计.md                (含 UI / UX / 视觉 / 产品设计)
```

## 应用顺序

判断用户的目标时,patterns 加载顺序:

```
1. 必读:通用.md
2. 必读:企业性质/<目标企业类型>.md
3. 必读:岗位/<目标岗位>.md
4. 加分:如果有岗位文件缺失,从相近岗位推断
```

**核心原则**:企业性质决定整体调性,岗位决定专业词汇。两者叠加用。

## 例子

**用户案例 A:应届生 → 字节 → 内容运营**
- 通用.md
- 企业性质/大厂.md(整体风格:量化、JD 匹配、敏捷)
- 岗位/运营.md(专业词:用户分层、内容策划、AB 测试)

**用户案例 B:社招 5 年 → 招行总行 → 财务**
- 通用.md
- 企业性质/银行.md(严谨、CPA、政治面貌)
- 岗位/财务.md(财务工具栈、专业证书)

**用户案例 C:留学生回国 → P&G → 市场营销**
- 通用.md
- 企业性质/外企.md(STAR、英文、Concise)
- 岗位/市场营销.md(Campaign 思维、消费者洞察)

## 每条 pattern 的标准格式

```markdown
### <一句话陈述模式>

- **理由**:为什么这样
- **示例**:(具体改写示例)
- **出处**:案例引用 / 行业惯例
- **适用条件**:(可选)
```
