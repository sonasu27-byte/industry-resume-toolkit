# 审计 - 改简历模式

针对"审计"岗位(进入会计师事务所方向)特有的改简历模式

---

## 模式

### 实习经历用"动作 + 落地 + 量化"三件套重写

- **理由**:审计岗位招聘官最看重的是"具体做过什么 + 做了多少 + 做的质量"。空泛描述等于没说
- **示例**:"底稿编制"这条改成:
  > "独立完成 1000+ 笔细节测试性抽凭工作;按照科目余额汇总表填写资产、负债、权益、损益类底稿。**通过细致的抽凭工作,确保账目与实际资产、负债数据的一致性,提升审计报告准确性。**"
  
  三件套:动作(独立完成 X 笔抽凭)+ 落地(按汇总表填底稿)+ 量化(具体数字)+ 一句"落地的成果"

### 个人职责用分号清晰拆步骤

- **理由**:审计工作流程性强,每条职责往往包含多个动作,分号能让招聘官看清你掌握的工作环节
- **示例**:"独立完成 X 笔抽凭工作;编制 XXX 底稿;按照科目余额汇总表填写资产、负债类底稿"

### 函证程序要量化"处理过的问题"

- **理由**:函证不是发函就完事,过程中处理过的具体问题(退函、拒回、客户拖延等)体现专业能力
- **示例**:"协助完成 X 家企业 300 银行询证函的编制、核对、发函、回函控制;确保函证过程高效,及时回应 X 位客户询问"

### 工具能力(Power BI / Python / SAP / SPSS)从项目里抽出来归到"计算机能力"

- **理由**:审计岗位越来越看重数据工具能力,集中放在技能区让招聘官一眼看到
- **示例**:沙盘项目里"利用 Power BI 进行数据采集"、"利用 Python 爬取数据" → 把工具名抽到技能证书的计算机能力,项目里只保留动作和量化

### 校内比赛(担任核心角色的)从"校园经历"挪到"项目经历"

- **理由**:经验较少的审计候选人,校内比赛(沙盘、案例大赛、大创、互联网+)如果担任核心成员或负责人,挪到"项目经历"详细写,能撑起整体经历厚度
- **示例**:大创/互联网+ 比赛(有获奖+核心成员)→ 从校园经历移到项目经历,按沙盘模拟比赛的详细写法补一段
- **适用条件**:实习≤1段,但参加过有获奖或核心角色的比赛

### 财务分析类项目要写明"具体哪些财务指标"

- **理由**:面试官会追问"你分析了什么指标",简历里就写清楚能省追问环节
- **示例**:"运用财务指标进行财务分析" → "运用 ROE、净利润率、毛利率、资产周转率等指标进行财务分析"
