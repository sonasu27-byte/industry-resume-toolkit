# 挖经历问题集:转行

**适用对象**:已有工作经验但要转岗位 / 转行业的人

**核心原则**:**只问事实,不问动机 / 规划 / 感受**。"为啥转 / 凭啥能做好"是面试问题,不写在简历里。简历只需要展示:**你过去做过什么 + 你为新方向做了什么具体准备**——这些都是事实。

**使用方式**:按轮次问,每轮答完才进下一轮。

> 注:用户的"目标岗位 + 行业 + 企业性质"已在 `/诊断` 流程里捕获,这里不再重复问。

---

## 第 1 轮:原岗位价值深挖(找 transferable skills 的事实根据)

> 不是问"你觉得你哪些能力可以迁移",而是问"你过去具体做过什么"——AI 自己判断哪些可迁移

1. 你目前 / 最近的岗位 + 公司 + 任期?
2. **业务理解力**:原岗位你深入懂哪些业务?(行业 / 客户 / 产品)
3. **数据 / 量化经验**:原岗位涉及什么数据?用什么工具?最大数据量级?
4. **沟通 / 协作经验**:跨多少团队 / 多少层级?主导过什么跨部门项目?
5. **项目管理经验**:有没有主导过完整项目?周期 / 团队规模 / 预算?
6. **客户 / 用户经验**:服务过多少客户?客户类型?

---

## 第 2 轮:具体业绩 / 成果(原岗位)

1. 你在原岗位**最自豪的 3 个成就**?(具体业务背景 + 你的角色 + 量化结果)
2. 拿过什么奖 / 优秀员工 / 季度 / 年度认可?
3. 带过新人吗?培养过几个?
4. 主导过什么内部 / 外部分享 / 培训 / 文档输出?
5. 离职原因?(简历开头一段或自我评价可能用到,但通常不写)

---

## 第 3 轮:新方向准备度

> **转行简历最关键的一节**——这部分都是事实,直接进简历

1. **自学过什么**?在线课程 / 书籍 / 训练营?(具体名称 + 时长)
2. **新方向相关证书**?(CFA / 数据分析师 / PMP / 教师证 / Coursera 证书)
3. **新方向副业 / 个人项目**?(做过什么 + 数据)
4. **新方向实习 / 兼职 / 志愿工作**?(哪怕短期 / 非正规)
5. **新方向社群 / 行业活动**?(进了什么群 / 参加什么活动 / 关注什么 KOL)

---

## 第 4 轮:跨界 / 隐藏经历

1. 你的**爱好 / 兴趣**有没有跟新方向相关的?(eg 程序员转产品,会做副业 app)
2. 你在原公司有没有做过**新方向相关的事**?(eg 销售岗做过内部产品 idea / 营销岗带过数据分析)
3. 你之前的**专业背景 / 学位**跟新方向有交集吗?
4. 你有没有**写过 / 表达过**对新方向的内容?(博客 / 公众号 / 朋友圈 / 知乎)

---

## 第 5 轮:技能盘点

1. 原岗位掌握的工具栈?(列全)
2. 这些工具里**哪些新方向也用得到**?
3. 语言能力(中 / 英 / 其他 + 具体水平)?
4. 学位 / 学校(尤其国内 211/985 / 海外学校在新方向是否有 buff)?

---

## 输出阶段

整理成 **`outputs/<用户>-素材库.md`**(纯事实,直接可写进简历):

```markdown
# <用户> 转行素材库

## 原岗位经历(精简后保留有迁移性的)
- [公司 + 岗位 + 任期 + 业务背景 + 量化成就]
- ⚠️ 标注哪些是 transferable skill 的事实根据

## 新方向准备(必须显眼写进简历)
- 自学 / 证书 / 副业 / 项目 / 实习
- **至少 2-3 条**,这是转行简历的核心 buff

## 跨界经历
- 兴趣 + 内部相关 + 学位交集 + 写作输出

## 技能 / 工具 / 语言
- 原岗位:[全部]
- 可迁移:[标出]
- 语言 / 学历:[具体]

## 自我评价拟句(纯事实)
- "X 年 [原行业] [岗位] 经验,具备 [可迁移能力 1 / 2 / 3],近期完成 [新方向准备 1 / 2 / 3],目标 [新岗位 + 行业]"

## 缺失项
- ⚠️ [项目 X 的数据]:用户没记清 → 必须补
- ⚠️ 新方向准备目前只有 [N] 条 → 建议接下来 [具体动作]
```

> **注意**:不要在简历里写"我想转行因为..."、"我相信能做好因为..."——这些是面试时口头讲的。简历只展示"我做过 X + 我准备了 Y"的事实。

把素材库写好后,**自动进入下一步:重组**。
