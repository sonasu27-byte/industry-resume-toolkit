# 示例:应届 + 审计

> **本案例为脱敏教学示例**。原型基于真实简历但所有姓名、公司、具体数据均已修改。
> 这份示例展示一份**经历相对简单**的应届简历如何应用本 toolkit 的三层框架优化。

---

## 简历背景

- **教育**:某 211 大学,会计学专业
- **实习**:1 段(国内某中型会计师事务所,中建系列项目)
- **项目**:沙盘比赛 + 案例大赛 + 创新创业比赛
- **校园**:学院红烛志愿者协会副部长(应急救援方向)
- **目标**:四大或国内大型会计师事务所审计岗

## 应用本 toolkit 的诊断结果

- **经历密度**:1 段实习 + 校园活动 + 比赛 → Phase 1 信息卫生 + Phase 3 经历挖掘需重点投入
- **目标企业性质**:外企(四大) + 银行(参考)
- **目标岗位**:审计 → 套用 `patterns/岗位/审计.md`

---

## 三层用力的分配

### Phase 1 信息卫生(重 — 格式问题是大头)

排雷清单:
- ✂️ **"备考 CPA、雅思"删** — 没考过就不写;考过部分科目写"CPA(已通过 XX 科目)"
- ✂️ **Office 不要拆开** — 不写"Word、Excel、PPT",直接写"Office"
- ✂️ **课程列表里的 Excel 删** — 放到"计算机能力"
- ⚠️ **教育背景成绩处理** — 只放 91、93、87 三门高分有问题。要么全放,要么不放
- ⚠️ **获奖荣誉位置错** — 应放"教育背景"课程下面另起一行,不单独立"校园经历-获奖荣誉"

### Phase 2 业务价值翻译(中 — 用三件套)

#### 实习段:某会计师事务所

**原文(空泛)**:
> "协助审计组完成中建四局下属公司的年审工作,负责底稿编制、函证程序、数据分析等工作。"

**改后(三件套)**:
> **底稿编制**:
> "独立完成 500+ 笔抽凭工作;按科目余额汇总表填写中建四局下属 2 家公司的审计底稿;**通过细致抽凭确保账实一致,提升审计底稿准确性。**"
>
> **函证程序**:
> "协助处理 12 家被审计单位的银行询证函;独立完成 300 封询证函的寄发与回函跟踪;**确保函证程序合规高效,函证回函率 95%。**"
>
> **数据分析**:
> "使用 SAP 系统处理 2000+ 条往来款数据;运用数据透视表分析关联交易;**通过数据可视化为审计判断提供决策依据。**"

(动作 + 落地 + 量化三件套)

#### 经历重新分类(经验较少型核心动作)

- 沙盘比赛获奖 → 移到"项目经历"详细写(原本在"校园经历"里被埋没)
- 大创/互联网+比赛 → 移到"项目经历"
- Python / Power BI / SPSS → 移到"技能区",从校园经历里挪走
- 学院奖学金 → 挪到"教育背景"课程下面

#### 社团活动用项目逻辑包装

红烛志愿者协会"消防观影日"按前期/中期/末期重组:

> **前期策划**:负责活动主题设计与流程规划;协调 3 个部门
> **中期执行**:主持 4 届活动,累计参与人次 800+;独立设计 60+ 道消防安全题
> **末期复盘**:整理 SOP 与素材库交接给下一届;形成可复用的活动手册

### Phase 3 经历挖掘(信息缺失自动触发)

经验较少型简历通常缺数据多。AI 检测到这份简历多处需要追问,会问 3-5 个最关键的问题:
- 立信实习:具体抽凭多少笔?编制底稿覆盖几家公司?
- 沙盘比赛:你的具体角色是?获奖了吗?获奖级别?
- 案例大赛:参考了哪些数据源?最终输出是什么?
- 大创/互联网+:具体担任什么角色?项目主题是什么?

用户答完 AI 填进去。不知道的标"建议补充",**不编**。

---

## 关键改造点(可复用)

1. **经验较少型先排雷再翻译** — Phase 1 不做好,后面白搭
2. **重新分类经历是核心** — 经验较少的人通常不是真没经历,是经历放错位置
3. **三件套(动作+落地+量化)** — 经验较少型每段实习/项目都要用
4. **社团活动按"前期/中期/末期"项目化** — 让校园经历看起来像项目
5. **经验较少型更需要 Phase 3 挖经历** — AI 会自动触发,问 3-5 个关键问题
